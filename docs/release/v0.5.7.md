# v0.5.7

## 新增功能 (Features)

- **代码重构与规范化**：
  - 将核心媒体逻辑 `media.go` 拆分为 `scanner.go` (扫描器)、`store.go` (存储层) 和 `media.go` (业务层)，提升代码可维护性。
  - 引入 CI/CD 流程 (`.github/workflows/check.yml`)，支持 push/PR 自动触发单元测试与 Lint 检查。
  - 引入代码质量工具配置 (`.golangci.yml`)，规范代码风格。
  - 修复 `internal/types` 中的弱类型定义，将 `Shares` 字段明确为 `[]config.Share`。
  - 补充核心模块 (`media`, `util`) 的单元测试，覆盖路径处理、文件分类、黑名单过滤等关键逻辑。

## 优化 (Optimizations)

- **稳定性提升**：通过强类型和单元测试覆盖，减少运行时错误的风险。
- **开发体验**：明确了模块职责边界，降低新功能开发的认知负担。

## 修复 (Fixes)

- 修复了部分潜在的空指针风险（通过 Lint 工具发现并修复）。
- 修复了 `config.Share` 类型定义在响应结构中的模糊性。

## 注意事项 (Notes)

- 本次更新主要涉及后端架构重构，不影响前端交互与配置兼容性。
- 推荐开发者在提交代码前运行 `go test ./...` 确保测试通过。
