# v0.6.0

## 核心变更 (Core Changes)

- **前端 UI/UX 全面升级**：
  - 遵循 UI/UX Pro Max 设计规范，全面改进用户界面和交互体验
  - 替换所有 emoji 图标为专业 SVG 图标（主题切换、排序按钮等）
  - 优化颜色对比度，提升文本可读性（浅色模式使用 slate-900/slate-600）
  - 改进所有交互元素的 hover 状态和过渡动画（统一为 200ms 平滑过渡）
  - 添加完整的键盘导航支持（`:focus-visible` 样式）
  - 优化响应式布局，改进移动端和小屏幕显示效果

## 新增功能 (Features)

- **图标系统升级**：
  - 使用 SVG 图标替代 emoji，提供更专业和一致的视觉体验
  - 主题切换按钮使用太阳/月亮 SVG 图标
  - 排序按钮使用箭头 SVG 图标，支持上下排序切换

- **交互体验改进**：
  - 所有可交互元素添加 `cursor: pointer`
  - 按钮 hover 时轻微上移并显示阴影效果
  - 列表项 hover 时边框高亮并轻微上移
  - 播放列表项 hover 时轻微右移，提供清晰的视觉反馈

- **无障碍性增强**：
  - 所有图标按钮添加 `aria-label` 属性
  - 支持 `prefers-reduced-motion` 媒体查询，尊重用户动画偏好
  - 改进焦点可见性，清晰的焦点指示器（2px 蓝色边框）

## 优化 (Optimizations)

- **视觉细节优化**：
  - 顶部栏添加轻微阴影，增强层次感
  - 对话框添加淡入动画，提升用户体验
  - 改进滚动条样式，更美观的滚动条设计
  - 标签页激活状态使用主色边框和文字，更清晰的选中状态

- **响应式设计**：
  - 优化移动端布局间距（640px 以下）
  - 顶部栏在小屏幕上自动换行
  - 改进对话框在不同屏幕尺寸下的显示

- **构建系统**：
  - 修复 PowerShell 构建脚本中 npm 调用问题
  - 改进构建脚本的兼容性和稳定性

## 修复 (Fixes)

- 修复构建脚本中 `& npm` 调用导致的 "Unknown command: 'pm'" 错误
- 改进浅色模式下的文本对比度，确保符合 WCAG 可访问性标准
- 修复浅色模式下边框可见性问题

## 开发者 (Developers)

- 所有 UI 改进遵循 UI/UX Pro Max 工作流程规范
- CSS 过渡动画统一为 200ms（符合 150-300ms 最佳实践范围）
- 使用 SVG 图标替代 emoji，提升专业性和一致性

## 注意事项 (Notes)

- 本次更新主要涉及前端 UI/UX 改进，不影响后端功能和 API 兼容性
- 所有更改向后兼容，现有配置和功能不受影响
- 建议用户清除浏览器缓存以获得最佳体验
